# Production Training and Evaluation

This script is designed to execute the final training run of the Large Language Model using the optimal hyperparameters identified during the Optuna search phase. Following training, it automatically saves the LoRA adapters and performs a comprehensive evaluation on the test dataset.

## Prerequisites

### 1. Configuration File

You must have the JSON configuration file generated by the Optuna search script. By default, the search script outputs a file named `*project_name*_best_config.json`.

Ensure this file is renamed to `best_config.json` or update the `BEST_CONFIG_PATH` variable in the script to match your specific filename.

**Example `best_config.json` structure:**

```json
{
    "r": 16,
    "lora_alpha": 32,
    "learning_rate": 0.0002,
    "num_train_epochs": 3,
    "per_device_train_batch_size": 4,
    "gradient_accumulation_steps": 2,
    "weight_decay": 0.01,
    "lr_scheduler_type": "cosine",
    "warmup_ratio": 0.05,
    "target_module": ["q_proj", "k_proj", "v_proj", "o_proj"]
}

```

## Usage

Execute the script via the command line:

```bash
python main.py

```

## Workflow Description

1. **Parameter Loading:** The script reads the JSON configuration file and maps the parameters to the training arguments. It automatically handles key discrepancies (e.g., mapping `target_module` from Optuna to `train_target_modules` for Unsloth).
2. **Model Initialization:** Loads the base model (e.g., `SmolLM2-1.7B`) in 4-bit quantization.
3. **Training:** Initiates the `SFTTrainer` using the optimal parameters. Metrics are logged to Weights & Biases under the project name `Final-Production-Training`.
4. **Artifact Preservation:** The trained LoRA adapters are saved to the `./final_model_lora` directory.
5. **Evaluation:**
* The model performs inference on the test dataset.
* It generates text based on the prompts.
* It parses the specific XML-like tags (`<score>`) to extract classification labels.
* It compares predictions against ground truth.


6. **Reporting:** A confusion matrix and classification report (Precision, Recall, F1-Score) are printed to the console and displayed.

## Output

* **Trained Model:** Saved in `./final_model_lora` (contains `adapter_config.json` and `adapter_model.safetensors`).
* **WandB Logs:** A new run in your WandB project containing the final training loss curves and system metrics.
* **Console Output:** Detailed classification metrics and the confusion matrix plot.
